### Confluent Cloud + Elasticsearch Setup

Free Trials:
* https://confluent.cloud/

Need to know:
* Examples use an already-setup Confluent Cloud account
* You need to generate API key for a cloud admin user to use terraform
* Since the example runs in a shared account, the environment is looked up rather than created
* Adjustment needed for your own environment
* Terraform will not run end-to-end in this example as we need to create flink jobs before we setup connectors. In my lab, I got around this by renaming `.tf` files to be `.tf.nouse` when I wanted to skip creating some terraform resources, which is why you will see the files split up in the `/terraform_cc` directory. I don't suggest this approach for a real-world use-case.


#### Step 1: Create terraform.tfvars

Copy the example and fill in your values:

```shell
cd terraform_cc
cp terraform.tfvars.example terraform.tfvars
```

Edit `terraform.tfvars` with your values:
- `cc_environment_name`: Your Confluent Cloud environment name
- `basename`: A unique name for your resources (e.g., "irops-demo")
- `elastic_endpoint`: Your Elasticsearch Cloud endpoint URL
- `elastic_username`: Usually "elastic"
- `elastic_password`: Your Elasticsearch password

#### Step 2: Apply Terraform

This will create:
- Kafka cluster in Confluent Cloud (Basic tier, ap-southeast-2)
- Topics: `irops_events`, `flight_manifest`, `hotel_inventory`, `transport_availability`, `enriched_irops_events`, `flight_schedule`
- Service account and API keys
- Flink compute pool (5 CFUs, ap-southeast-2)
- Elasticsearch Sink Connectors (2 connectors)

```shell
cd terraform_cc
terraform init
terraform apply
```

Review the plan and type `yes` to confirm.

#### Step 3: Create .env file

```shell
terraform output -raw dot_env > ../loader/.env
```

#### Step 5: Register JSON Schemas (Required for Flink)

Your test data was produced as raw JSON strings. To use Flink SQL with Schema Registry, you need to register JSON Schemas first.


**Quick steps via Confluent Cloud UI (copy-paste):**
1. Go to Schema Registry in Confluent Cloud
2. Click **Add schema**
3. For each schema file in `schemas/`:
   - Subject: `{topic-name}-value` (e.g., `irops_events-value`)
   - Schema type: **JSON**
   - Copy-paste schema content
   - Click **Create**

**Schemas to register:**
- `schemas/irops_events.json` → subject: `irops_events-value`
- `schemas/flight_manifest.json` → subject: `flight_manifest-value`
- `schemas/flight_schedule.json` → subject: `flight_schedule-value`
- `schemas/hotel_inventory.json` → subject: `hotel_inventory-value`
- `schemas/transport_availability.json` → subject: `transport_availability-value`

This enables **"schemaless JSON"** mode where Flink uses Schema Registry to parse raw JSON data.


### Step 6: Set Up Flink SQL

We are doing 3 things with flink:

#### I - Flink enriches IROPS events with passenger manifest data

File: `sqls/enriched_irops.sql`

**Before Flink:**
```json
{
  "event_id": "IROPS_TB519_001",
  "flight_number": "TB519",
  "status": "DELAYED"
}
```

**After Flink:**
```json
{
  "event_id": "IROPS_TB519_001",
  "flight_number": "TB519",
  "status": "DELAYED",
  "vulnerable_passengers": [
    {"pnr": "ZX91KL", "name": "Margaret Chen", "ssr_codes": ["WCHR"]},
    {"pnr": "QW77YU", "name": "Sophie Nguyen", "ssr_codes": ["UMNR"]}
  ],
  "connection_passengers": [
    {"pnr": "LM44RT", "name": "James Patel", "onward_connection": "SIN"}
  ],
  "total_passengers": 50,
  "vulnerable_count": 12
}
```

#### II - Flink updates flight schedule with IROPS events

File: `sqls/schedule_vs_actual.sql`

**Before Flink:**
```json
{
    "flight_number": "TB519",
    "airline": "TrueBlueAustralia",
    "aircraft_registration": "VH-TBA",
    "aircraft_type": "B737-800",
    "origin": "SYD",
    "destination": "BNE",
}
```

**After Flink:**

```json
{
    "flight_number": "TB519",
    "scheduled_destination": "BNE",
    "scheduled_arrival_time": "2026-05-26T07:30:00Z",
    "actual_landing_airport": "OOL",
    "actual_landing_time": "2026-05-26T12:30:00Z",
    "arrival_delay_minutes": 300,
    "arrival_status": "2026-05-26T12:30:00Z",
}
```

#### III - SMS notifications

Flink generates new messages, like this. These would then need to be enriched with phone number to allow sending.

File: `sqls/sms_notifications.sql`

```json
{
    "event_id": "IROPS_TB519_006",
    "pnr": "PL77DF",
    "flight_number": "TB519",
    "passenger_name": "Noah Taylor",
    "onward_connection": "LAX",
    "notification_type": "CONNECTION_AT_RISK",
    "message": "URGENT: Flight TB519 is diverting. Your connection to LAX may be affected. Please contact airline staff upon landing."
}
```
